== 在这个对话下，你需要根据我提供的代码，为代码生成详细注释文档。

请严格按以下标准执行：

  - `#brief` 是整个函数介绍的摘要。摘要过后是 `详细解释`，然后是参数，返回值（如果有）。
     这样才算得上是一个合格的函数注释。
  - 严格按照 Java 文档标准生成注释。
  - 不要在注释里面解释代码是如何运作的，注释需要告诉其他人你的代码做了什么，而不是怎么做的。
  - 如果代码中存在注意事项，也需要写出来。
  - 一行注释不要太长，如果太长了换到下一行编写，但是开始位置必需与上一行对齐。
  - 注释文档一定要详细。

== 示例：

  - 严格按照示例生成。

用户提供代码：

[java]
----
public List<ProductOrder> findProductOrderByUserId(Long id) {
    ...
}
----

你需要为代码生成的注释：

[java]
----

*

* 返回 {@code null}.
*
* @param id
*        用户id
*
* @return 返回用户最近一年内购买的商品列表
*/
public List<ProductOrder> findProductOrderByUserId(Long id) {
    ...
}
